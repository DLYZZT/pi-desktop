# Changelog

All notable changes to Pi Agent Desktop are documented in this file.

## [0.1.1] - 2026-07-17

### Fixed

- Show model request failures directly in the conversation instead of silently rendering an empty assistant reply. Provider error details such as an invalid API key response are preserved, with an actionable fallback when the provider returns no detail.
- Keep failed assistant messages visible after conversation grouping and reload while continuing to hide harmless empty responses.
- Add regression coverage for provider failures, missing error details, and normal empty assistant messages.

### Changed

- Run tests and the full quality gate on pushes to `main` without packaging temporary installers.
- Keep temporary package artifacts available for pull requests and manually dispatched workflows.
- Add the Windows x64 NSIS installer, blockmap, and update metadata to tagged desktop releases alongside signed and notarized macOS artifacts.
- Strengthen release asset validation for both macOS and Windows packages.
- Update the Chinese and English release documentation and ignore the local `temp/` directory.

### Platform notes

- macOS release packages are signed and notarized. Stable in-app updates use the GitHub Release metadata generated by the release workflow.
- The Windows installer is a production release asset but is not currently Authenticode-signed. Windows may show an unknown-publisher or SmartScreen warning during installation.
- In-app updates remain disabled on Windows. Windows users should download and install new versions manually from GitHub Releases.
